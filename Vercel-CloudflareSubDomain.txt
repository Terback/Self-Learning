这是一个非常典型的跨平台域名配置场景：你的域名解析权（DNS）在 Cloudflare，但你的 Dashboard 网页托管在 Vercel。

按照以下步骤操作，可以确保 hub.eimtechnology.com 快速平稳地上线。

第一步：在 Vercel 中添加自定义域名
登录 Vercel Dashboard，进入你新创建的那个 Dashboard 项目。

点击顶部的 Settings 选项卡。

在左侧菜单中选择 Domains。

在输入框中输入 hub.eimtechnology.com，然后点击 Add。

此时 Vercel 会显示一条警告，提示 Invalid Configuration。别担心，它会给你两个关键信息（选其一即可，推荐使用 CNAME）：

Type: CNAME

Name: hub

Value: cname.vercel-dns.com


第二步：在 Cloudflare 中配置 DNS
登录 Cloudflare 控制台，点击进入 eimtechnology.com 这个域名。

点击左侧菜单的 DNS -> Records。

点击 Add record（添加记录）：

Type: 选择 CNAME。

Name (required): 输入 hub。

Target (required): 输入 cname.vercel-dns.com。

Proxy status: 重要！ 请先关闭橙色小云朵，将其设置为 DNS Only（灰色）。

注：Vercel 本身自带 CDN 和 SSL 加密。虽然 Cloudflare 代理也能用，但初次配置时设为 DNS Only 能让 Vercel 更快检测到解析并签发 SSL 证书。等生效后，你随时可以再开启橙色云朵。

点击 Save。


第三步：回到 Vercel 验证
返回 Vercel 的 Domains 页面。

点击 Refresh 或者等待几分钟。

当看到 Valid Configuration 的绿色对勾，且下面的 SSL 状态变为 Active 时，说明配置成功。


第四步：Cloudflare SSL/TLS 设置检查（避坑指南）
如果在访问时出现 Too many redirects（重定向次数过多）的错误，请检查 Cloudflare 的 SSL/TLS 设置：

确保你的加密模式至少是 Full（完整）或者 Full (Strict)。

原因： Vercel 强制使用 HTTPS，如果 Cloudflare 设置为 "Flexible"，Cloudflare 会用 HTTP 去请求 Vercel，Vercel 又重定向回 HTTPS，从而形成死循环。

进阶建议（针对你的 Vibe Coding）
既然你已经有了好几个子域名（invoice, teamschedule 等），建议你在 Cloudflare 里把这些 CNAME 记录统一管理：

Vercel 项目： 全部指向 cname.vercel-dns.com。

Vercel 部署： 以后如果你有新的小工具，只需在 Vercel 绑定域名 + Cloudflare 增加一个 CNAME 即可。
